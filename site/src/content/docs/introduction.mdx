## What is 1ls?

1ls (pronounced "one-less") is a lightweight command-line tool that processes structured data using JavaScript syntax. It automatically detects and parses various file formats including JSON, YAML, TOML, CSV, and more - then lets you transform the data using familiar JavaScript.

## Why Use 1ls?

### The Problem

Working with different data formats on the command line often requires multiple tools:
- `jq` for JSON
- `yq` for YAML
- `csvkit` for CSV
- `tomlq` for TOML

Each tool has its own syntax and quirks to learn.

### The Solution

1ls handles all common data formats with one tool and one syntax - JavaScript:

```bash
# JSON
echo '{"name":"Alice"}' | 1ls 'x.name'

# YAML
cat config.yaml | 1ls 'x.database.host'

# TOML
cat Cargo.toml | 1ls 'x.dependencies'

# CSV
cat data.csv | 1ls 'x.filter(row => row.age > 21)'

# Package.json
1ls 'x.scripts' < package.json
```

## Supported Formats

1ls automatically detects and parses:
- **JSON** - JavaScript Object Notation
- **YAML** - YAML Ain't Markup Language
- **TOML** - Tom's Obvious, Minimal Language
- **CSV** - Comma-Separated Values
- **TSV** - Tab-Separated Values
- **XML** - Extensible Markup Language
- **INI** - Configuration files
- **Plain text** - Line-by-line processing

## Benefits

- **Universal**: One tool for all data formats
- **Familiar Syntax**: Just JavaScript, no new query language
- **Auto-detection**: Automatically identifies file format
- **Lightweight**: Minimal dependencies, fast execution
- **Unix-friendly**: Works seamlessly with pipes
- **Fast**: Built on Bun for optimal performance
